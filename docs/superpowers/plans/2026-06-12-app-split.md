# Split di `app.js` — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Spezzare `app.js` (4123 righe) in moduli a confini netti, un'estrazione alla volta, **senza cambiare comportamento**, fino a portarlo a ~1500 righe (resta il solo cuore "focus").

**Architecture:** ESM diretto nel browser (nessun bundler). `app.js` resta entry/orchestratore e tiene i suoi `let` di stato invariati; un modulo `app-context.js` espone quei local come **proprietà accessor vive** su un oggetto `ctx`. I moduli estratti leggono/scrivono `ctx.data`, `ctx.openIndex`, … e fanno re-render via `ctx.render()`. Le chiamate di lancio fra overlay con stato passano per `ctx` (mai import diretto fra moduli con stato); i leaf utility senza stato si importano direttamente.

**Tech Stack:** JavaScript ESM, Node.js test runner (`node --test`), Playwright (`npm run e2e`), Service Worker con precache list (`sw.js`).

**Spec di riferimento:** `docs/superpowers/specs/2026-06-12-app-split-design.md`

## Convenzioni e Ricetta di estrazione

**Branch:** lavora su `refactor/app-split` (fai `git fetch origin` + sync con `origin/main` PRIMA di iniziare — su questo repo il main locale può essere indietro). ff-merge + push **a fine di ogni ondata** (4 checkpoint).

**Identificazione del codice:** i task indicano le funzioni **per nome** e per marcatore di sezione (es. `// ---- Calendario`), NON per numero di riga assoluto: le righe scorrono man mano che si estrae. Usa `grep -n "function nomeFunzione"` per localizzarle nel file corrente.

**Behavior-preserving:** zero cambi di logica. Il diff di ogni estrazione è: (1) taglia codice da `app.js`, (2) incolla nel nuovo modulo, (3) ripunta i riferimenti allo stato condiviso a `ctx.*` **solo nel codice spostato**, (4) aggiungi `import`/`export`, (5) ricabla in `app.js`.

### RICETTA (i passi standard di ogni task di estrazione)

Per il modulo `<M>.js` con la lista di simboli del task:

1. **Crea `<M>.js`** con l'header di import indicato; **sposta** le funzioni e lo stato privato elencati (taglia da `app.js`).
2. **Ripunta** nel codice spostato: stato condiviso → `ctx.x`; helper trasversali (`planDays`, `fmtKg`) → import da `./app-context.js`; chiamate ad **altri overlay con stato** → `ctx.openX()`; leaf utility/puri → import diretto.
3. **`export`** i punti d'ingresso elencati.
4. **In `app.js`:** rimuovi il codice spostato; aggiungi `import { … } from "./<M>.js"`; assicurati che `render()` chiami i renderer importati e che il blocco di wiring `ctx` registri i punti d'ingresso richiamati da altri overlay.
5. **In `sw.js`:** aggiungi `"./<M>.js"` all'array `ASSETS` e bumpa `CACHE` al valore indicato dal task.
6. **Verifica:** `node --test` (≥ 446 pass, 0 fail) **e** `npm run e2e` (boot online+offline verdi).
7. **Commit:** `git add -A && git commit -m "refactor(app): estrai <M>.js da app.js"`.

Se durante l'estrazione scopri che una funzione spostata chiama un'altra funzione **non ancora estratta che vive in `app.js`**: registrala su `ctx` in `app.js` (`ctx.fnName = fnName`) e chiamala come `ctx.fnName()` nel modulo. Se chiama un **leaf utility o modulo puro**, importalo direttamente.

- [ ] **Step 1: Crea `app-context.js`**

```js
// app-context.js — stato condiviso fra app.js e i moduli estratti.
// I campi di stato (data, currentWeek, …) sono definiti come accessor da app.js
// (bridge), così leggono/scrivono i `let` di app.js senza duplicare lo stato.
import { PLAN } from "./plan.js";

export const ctx = {
  render: () => {},   // sovrascritto da app.js: ctx.render = render
};

export const planDays = () => (Array.isArray(ctx.data.plan) && ctx.data.plan.length ? ctx.data.plan : PLAN);
export const fmtKg = (n) => Math.round(n).toLocaleString("it-IT");
```

- [ ] **Step 2: Aggiungi import + bridge in `app.js`**

Aggiungi l'import insieme agli altri in cima a `app.js`:
```js
import { ctx } from "./app-context.js";
```
Subito **dopo** il blocco delle dichiarazioni `let` di stato (dopo `let pusher = null;`, ~riga 65), inserisci:
```js
// Bridge: espone i local di app.js come proprietà vive di ctx, così i moduli
// estratti usano ctx.<x> senza che app.js cambi i propri riferimenti.
Object.defineProperties(ctx, {
  data:           { get: () => data,           set: (v) => { data = v; },           configurable: true },
  currentWeek:    { get: () => currentWeek,    set: (v) => { currentWeek = v; },    configurable: true },
  currentDay:     { get: () => currentDay,     set: (v) => { currentDay = v; },     configurable: true },
  openIndex:      { get: () => openIndex,      set: (v) => { openIndex = v; },      configurable: true },
  nutritionOpen:  { get: () => nutritionOpen,  set: (v) => { nutritionOpen = v; },  configurable: true },
  planOpen:       { get: () => planOpen,       set: (v) => { planOpen = v; },       configurable: true },
  store:          { get: () => store,          set: (v) => { store = v; },          configurable: true },
  session:        { get: () => session,        set: (v) => { session = v; },        configurable: true },
  profileStorage: { get: () => profileStorage, set: (v) => { profileStorage = v; }, configurable: true },
  dataVersion:    { get: () => dataVersion,    set: (v) => { dataVersion = v; },    configurable: true },
  pusher:         { get: () => pusher,         set: (v) => { pusher = v; },         configurable: true },
});
```
E dopo la definizione di `function render()` (o in fondo, comunque dopo che `render` è dichiarata — le function declaration sono hoisted, quindi puoi metterlo subito sotto il bridge):
